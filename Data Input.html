Best Practices with Data Input and Visualization (Python) Part 1¶

A lot of data science is about creating hypotheses and then testing their validity against a set of data. In this context, the fundamental points of data science are all focused around cleaning and visualizing data in order to have adequate hypotheses to test. Another misconception with the job of a modern data scientist is assuming that their job is about designing highly specific algorithms. In reality, their job is about implementing already well established ML algorithms(usually well implemented in major scientific libraries) based on the contextual understanding they build from data analysis.

All of this is to elucidate the point that data input, analysis, and visualization are some of the most critical parts of a data analyst's day to day job. The purpose of this post is to clarify some of the best practices for data analysis that I've come across, specifically with python libraries.

Input from Text Files¶

Often times, you're going to get data as an ASCII file. Data analysis without checking data in this format is easily corruptible due to simple things such as: different separators between data entries (spaces vs tabs), empty lines at the end of the file, etc.

This warrants a cursory visual inspection of whatever data you are daeling with. Some things that I always check are:

  • Does the data have a header/footer?
  • Are there any empty lines at the end of the file
  • Are there any white spaces before or after data entries?
  • Tabs or spaces to separate data entries?

Use a IDE or editor that allows you to visualize white spaces, tabs, or spaces in order to ensure this.

Reading ASCII data into Python¶

One of the best (if not the best) way to analyze and visualize data quickly is to use IPython notebooks (like this one). It allows you to quickly read in, analyze and manipulate data without too much setup. Once you're done, you can easily export your python notebook command history to any IDE or editor using %history command. In python, loading in data is extraordinarily simple. You just utilize one of the methods of existing libraries. One can use numpy.loadtxt, but its easier (and more beneficial long term a quick workflow of the steps you utilize.

  • 1) import pandas as pd (can be anything, pd is a common name)
  • 2) find your data file in your local storage (or online storage)
  • 3) file = "data.txt"
  • 4) df = pd.read_csv(file)
  • 5) (this is how you read the file in using pandas. Pandas has an inbuilt object called a dataframe, which is a 2d data structure that is like a matrix object with many inbuilt methods)
  • 6) df.head() #one of the in-built methods. Displays the first line of the data.
  • 7) df.tail() #displays the last line

last two lines are usually good starting points to ensure the beginning and end of the data is what you expected. Also allows you to check how many columns and rows there are. Can also use the shape method to check the overall size of the dataframe, and generally see if there were any glaring errors with reading in your file (usually not the case).

Regular Expressions.¶

Regular Expressions (Regex) are a type of minor scripting language (supported by many large scale languages like Python, R etc) that is very useful for data analysis as it allows for effective text pattern matching. Sometimes with complex or real world data sets, you will find values in disarray. Sometimes they will be separated by some combination of commas, semicolons, and spaces.An example of this would look something like:

  • df= pd.read_csv(file, sep='[;,]+')

In this case, the square brackets indicate that the data may be separated by a combination of a comma and a semicolon and a plus indicates one or more of each. That is a regex expression (which the square bracket indicates). It allows the content inside the separator parameter to run as a minifunction that separates data based on some combination of commas and semicolons.

Below I'll demonstrate more about regex by extracting a column from a random data set generated through the numpy random number generator.

Making Dataframes from Matlab and Excel¶

Matlab¶

Matlab support is built into the scipy library. The command is:

  • from scipy.io load loadmat
  • data = loadmat('data.mat')

Excel¶

There are two basic approaches to loading data from excel files (both in numpy)

  • 1) read_excel- for excelbooks that have identical data formats across internal data sheets.
  • 2) ExcelFile - for excelbooks that have different data formats across internal data sheets.

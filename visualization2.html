Data Input and Visualization Best Practices Part 2¶

Data Visualization with Python and Jupyter Notebooks¶

Displaying and visualizing statistical data is one of the most fundamental portions of a data science job. The logic behind this pertains to how adept humans are at visual processing. We are more capable of spotting irregularities, and extreme data values, potentially correcting for outliers. We are also able to form better hypotheses by naturally finding obvious visual patterns.

Data Types for Statistical Analysis¶

The statistical procedure we use depends on the type of data we have. One way we can divide data is into classification type, i.e. categorial or numerical data. Another way we can divide data is into number of paramaters/variables it has: i.e. univariate, bivariate, or multivariate.

Categorical Data¶

We'll start with Categorical Data. As the name implies, this refers to a data type that reflects categories. We can break categorical data down further into 3 parts.

  • Boolean Data- data which can only have two values. E.g., True/False, female/male, drinker/non-drinker.
  • Nominal Data- data which can have more than two values but is not ordered. E.g. Married/Single/Divorced
  • Ordinal Data- data which can have more than two values but has a logical sequence (ordered values) e.g. few/some/many.

Numerical Data¶

Numerical data is self explanatory as well. We can break numerical data further into 2 categories.

  • numerical continous- data which reflects a smooth function.
  • numerical discrete- data that can only take on real integer values. Something like counting a number of elements in a set

Plotting in Python¶

We can generate plots with iPython notebooks with either matlab-like (using matplotlib) style or also in an object-oriented style that is more suited in the python style (using objects). I'll show both below (though I prefer matlab like styling). In my understanding, its better to stick to one style throughout your code, but beyond that it comes down to personal preference for simpler plots. For complex plots, especially when you need more interactivity, its better to use python style objects.

Below we'll start with the pyplot, matlab style.

In [6]:
import matplotlib.pyplot as plt 
import numpy as np 

#we're just going to use arange to generate some data 
x = np.arange(0,10,0.2)
y = np.sin(x)

plt.plot(x,y)

plt.show() #this takes care of the axes, and graph creation without you worrying about details.
In [7]:
# now the same thing as a python object. You'll notice an object address is shown above the plot. 
fig = plt.figure()
ax = fig.add_subplot(111)
ax.plot(x,y)
Out[7]:
[<matplotlib.lines.Line2D at 0x10e7c10b8>]

Below we'll show why interactivity and python like object plotting is more useful, as you add more and more functions to a plot. We'll continue using the x and y we defined above, but will now introduce a third variable z.

In [16]:
z = np.cos(x)

fig, axs = plt.subplots(nrows=2, ncols=1) #generating figures and our axes
#on the first axis, we plot sine 
axs[0].plot(x,y)
axs[0].set_ylabel('Sine')

#on the second axis, we plot the cosine
axs[1].plot(x,z)
axs[1].set_ylabel('Cosine')

plt.show()

Kernel-Density-Estimation (KDE) Plots¶

Histograms are useful at a superficial level, but two of their biggest flaws are that they are

  • discontinous
  • shape depends on the chosen bin-width (interval size)

In some cases, we would prefer to use KDE (Kernel-Density-Estimation) Plots. We will not go too into the statistics here, as this post is solely about visualization tools, but the KDE utilizes a special version of a PDF (a Kernel) to fix the issues addressed above.

the way that you use a KDE in python is shown below.

In [26]:
x = [-2.1, -1.3, -0.4, 1.9, 5.1, 6.2]
sns.kdeplot(x)
Out[26]:
<matplotlib.axes._subplots.AxesSubplot at 0x1a11e40080>

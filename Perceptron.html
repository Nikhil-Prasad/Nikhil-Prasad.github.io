Perceptron Learning Algorithm

Posted on Sat 19 May 2018 in machine Learning

Perceptron Learning Rule- ML Algorithms Part 1¶

This is an implementation of the perceptron learning algorithm, which was the first computatutational represenation of a neuron (1957). In the modern sense, its only really useful as a binary classifier and interesting as the starting point for the field of neural networks.

The perceptron algorith is a classifier. Broadly speaking, all classification problems can be broken down into 2 distinct types, linearly separable or non-linearly separable. Distinction between these two is whether the data can be classified into two groups by a hyperplane. In one variable one target situations, this just means that the data can be separated by a line. Some simple examples of "linearly separable data" are the boolean function categories. That is to say, the perceptron can model data that fits into the logic of "and", "or" (by drawing a line through their truth values). In this specific example we'll be using the famous Iris data set, which clearly categorizes to "or" i.e. the flower is either this or that. Another dataset example that we know would be linearly separable is "and" i.e. either the data cotains both of these features or it doesn't.

How does the Perceptron Algorithm work?¶

The perceptron algorithm works basically by "iterating" through finding the hyperplane that best distributes the data. Since this is a supervised learning method (probably the most basic supervised learning method), it "iterates" by comparing some output that it generates against the actual output. Basically the perceptron starts with a "hyperplane". In 2-dimensions it starts with a line. This line is initalized at an arbitrary point (explained in the comments of the code). Some data (x,y) type, is fed into the algorithm. The algorithm uses a function to compute a classification. The way it does that is to sum through all the data classifcation input points with some weight to heaviside function in this case) to compute a value of either +1 or -1. This is how the perceptron initially classifies data. It then compares its own classification with the actual output that it gets from the training data, and uses that error term to change its own classification by updating the weights that it gave each input with that error term.

Perceptron Class¶

Below we define the perceptron class.

In [7]:
"""
Perceptron. Binary classification. Have an activation function with a threshold. Binary classification maps to this 
using heaviside function. Activation function is dot product of weight transposed and input variable data. 
If above threshold, gets transformed to +1, if not -1. Compared against actual output (training data). Weights updated
from there if output doesn't match. Continues passing through until there are no more updates in the weight matrix. 
"""
import numpy as np 
class Perceptron(object):
    """Perceptron Classifier 
    Parameters 
    ---------
    eta : float 
        Learning rate (between 0.0 and 1.0)
        n_iter = int 
            iterations over training dataset 
    Attributes 
    ---------
    w_ : 1d array 
        Weights after fitting 
    errors_ : list 
        Number of misclassifications in each iteration/passthrough/epoch
    """

    def __init__(self, eta=0.01, n_iter = 10):
        self.eta = eta
        self.n_iter = n_iter 
        
    def fit(self, X, y): #loops over individual samples in training set and updates weight matrix. 
        """ fit training data 
        
        Parameters 
        X: {array-like}, shape = [n_samples, n_features]
        Training vectors, where n_samples is the number of samples and n_features is number of features 
        y: array-like, shape = [n_samples]
        Target values 
        
        Returns 
        -------
        self : object 
        """
        self.w_ = np.zeros(1 + X.shape[1]) #initialize weights matrix with zeros + the bias term 
        self.errors_ = [] #errors matrix. 
        
        for _ in range(self.n_iter): #loop through all the epochs 
            errors = 0 
            for xi, target in zip(X, y): #zip makes a tuple out of Xi,target entries. iterate through. 
                update = self.eta * (target - self.predict(xi)) #learning rate * actual output - predict function for xi
                self.w_[1:] += update *xi #updates correspodining weights for passthrough
                self.w_[0] += update #changes the w_0 bias term
                #print(self.w_) wanted to see how weights matrix changed through passthroughs
                errors += int(update != 0.0) 
            
            self.errors_.append(errors)
        return self 
    
    def net_input(self,X):
        """ Calculate net input """
        
        return np.dot(X, self.w_[1:]) + self.w_[0] #activation function dot product. 
    
    def predict(self,X):
        """Return class label after unit step """
        return np.where(self.net_input(X) >= 0.0, 1, -1) #if input >0, return 1 else -1. Which is the heaviside function

In the code below, we're going to create a perceptron class object called ppn, with a learning rate of 0.1 and ask it to go through 10 iterations of the data.

Learnign rates are values between 0 and 1 that are multipied by the error before adding it to the new weights. This can be thought of as a "dampening" effect on the rate of improvement. The goal is to restrict how quickly the algorithm learns from 1 or 2 datapoints because it will oscillate too wildly and perhaps get stuck in a local maxima or miss the convergence point (i.e. swing away from the actual line of fit). You want it to converge slowly onto an optimization (a line that actually separates the data).

So we then use this ppn object to fit the iris data. I then want to begin by plotting the errors function over all 10 iterations of the data to show how the errors eventually converge to 0.

from matplotlib.colors import ListedColormap

Now we want to create a new function, called plot decision regions, that takes a classifier object such as the pereptron and "divides" the plot against the classifier object. This is just going to be used to show that the perceptron object actually did its job, by plotting it against the data.

The function first generates a plot through matplotlib's listedcolormap method, then creates a range based on the input data. It then takes the inputted classifier and uses its predict method based on the inputted data. (in this case the perceptron, but could be any classifier object, in the Adaline post, I will use the same function with an adaline classifier). It then reshapes and plots the decision line/hyperplane and then plots the class samples.

In [14]:
def plot_decision_regions(X,y,classifier, resolution=0.02):
    #setup marker generator and color map 
    
    markers = ('s', 'x', 'o', '^', 'v')
    colors = ('red', 'blue', 'lightgreen', 'gray', 'cyan')
    cmap = ListedColormap(colors[:len(np.unique(y))])

    #plot the decision surface based on input data.
    x1_min, x1_max = X[:, 0].min() - 1, X[:,0].max() + 1
    x2_min, x2_max = X[:, 1].min() - 1, X[:,1].max() + 1
    
    xx1, xx2 = np.meshgrid(np.arange(x1_min, x1_max, resolution), np.arange(x2_min, x2_max, resolution))
    
    #classifier predicts based on 1d Arrays of the input data. 
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1,xx2,Z,alpha=0.4, cmap=cmap)
    plt.xlim(xx1.min(), xx1.max())
    plt.ylim(xx2.min(), xx2.max())
    
    #plot class samples 
    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(x=X[y == cl, 0], y=X[y == cl, 1], alpha = 0.8, c=cmap(idx), marker = markers[idx], label=cl)

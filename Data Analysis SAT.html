Data Cleansing Exercise- SAT Scores- Prep for Predictive Loan Analytics Pt. 1¶

The purpose of this exercise is to gain more experience in data cleansing and data engineering. The data is taken from NYC open data, and the analysis goes through the steps of understanding the data fully, integrating multiple datasets together by finding commonalities in the data, condensing the data by doing basic feature engineering, and then computing correlations to gain some context around what factors seem to influence SAT scores in new york city. This is based on a dataquest blog post about a kaggle competition regarding SAT scores in New york City, and how to properly approach the process of data cleaning. This is in preparation for a future college loan project predictor that I will be building. We'll end up using the usual suspects; pandas, numpy, and matplotlib to do this data cleansing.

The first thing that I notice is that some of the data sets contain multiple rows for each high school, such as class size. This makes sense, as each school may have up to 6 class sizes if it is elemntary, 2-3 if it is middle school, and 3-4 if it is a high school. ALl of those would be listed under the same high school name. This is something however that jumps out as not being useful for our purposes, as we want each school to have one row for ease purposes. This will require us to do some data preprocessing later on. We also see that most of the datasets have a DNB column. This seems like a unique school identifier. Looking it up, we see that it stands for District Borough Number, and that it is infact a unique identifier for each school. It could be a value that we use to organize and unify the data. Since we know the DBN value for each school name, we can add it to the datasets that don't have it pretty easily. In the following step, we add the DBN values to class_size and HS directory. In HS directory, the dbn column exists but is lowercase, so we just rename it to DBN.

If we look through class size, we see that the DBN identifier has been broken into its 3 separate components in that dataset. DBN is made up of the district, the borough, and the school code. if we look at class size by itself, as I do below, we can see that all 3 exist as separate columns. we can create a new column in class_size by concatenating these 3 column values.

Looking at class size dataset, we see the school Roberto Clemente has repeats in Grade and Program Type. Grades makes sense, as we explained earlier, and so does Program_type, as it seems to indicate, special education, general education etc. We know that we have to restrict each field to a single value, as ultimately the program types and grade types create duplicate rows. This will interfere with total class_size however; we fix that by computing class_size averages for each school and using that average for our overall correlations (this assumes that each class within each school is about the same, but that seems a fair assumption)

Combining Data¶

We now combine the data using the DBN column as a unique identifier. The end result will be a dataset with tons of columns from all the original data sets. If there are potential data sets with missing columns (in this case there are not) We could resolve this by merging datasets with missing rows using outer join, which will basically give us a "union" of all the datasets, with null values in the datasets that don't have those specific columns. In the data used in the blog post, there was one missing column in AP_results. The data that I took from NYC open data does not have that missing value.

In the code below, we loop through all the items (datasets) in the data dictionary we created at the beginning. If the item column value is in one of the 3 listed datasets (sat_results, ap, graduation), we do an outer join (union) if it is not, then we merge into the full dataset object.

Adaptive Linear Neuron (ADALINE)

Posted on Mon 21 May 2018 in posts

Adaline (ADAptive LInear NEuron). ML Algorithms Pt 2.¶

Adaline is one of the earliest single layer neuron implementations for binary classification. ( I worked with the perceptron in the previous ML from Source post). The key difference between Adaline and the Perceptron are in the weight functions. In the perceptron, the sum of the weights and inputs (dot product) are updated using the heaviside function, that is, the outputs above a certain threshold are "transformed" to +1 or -1, thereby forming the structure for classification. In adaline, the weights are adjusted according to the actual sum of the outputs, instead of being transformed to a binary classification scheme.

In most cases in supervised learning (unlike Perceptron) the goal is to pick a representative function that we want to optimize during the learning process. This representative function is often a cost function that we want to "minimize". In the case of Adaline, we define the cost function J to learn the weights as the "sum of squared errors" between the predicted outcomes and the actual class labels.

One of the main advantages that we get from having this continous linear activation function as the cost function, opposed to the heaviside function is that it becomes differentiable and convex. Convexity mathematically says that the 2nd derivative of said function will always be greater than zero; visually it says that any line drawn between two points on the function graph will aways cross over it.

These properties allow us to use an optimization algorithm known as gradient descent to find the weights that are necessary to minimize the cost function (Sum of Squared Errors). Sum of squared errors is just the sum of the error/difference/residual between the actual classificaiton values of the outputs and our predicted values. By using gradient descent to minimize this function, we are naturally finding the optimal weights for our dataset.

How does Gradient Descent Work?¶

If we take any n-dimensional function F(x), and assume that it is defined and differentiable around some point a, then the negative gradient of this function around point a is the "steepest descent" possible. Gradient descent takes advantage of that by startin at a point a, and "moving" a small distance in the direction most negative of the gradient (unit vector in the direction opposite of the gradient). It then reaches some new point a1, checkes the gradient again, and proceeds in the direction of the most negative gradient. It continues doing this until it reaches a point where there is no negative gradient, implying that it has reached a significant local minima.

An analogy I use for gradient descent is to imagine trying to climb down a hill in pitch black with a flashlight that can only see a few feet ahead. Your strategy to move down the hill could be to shine the flashlight around you, find the steepest descent point you see, take a few steps, and repeat the process. If you reached a point where you were surrounded by only positive gradient or zero slope, you'd know that you either reached some sort of saddle or crevice in the hill, or reached the bottom. This depends completely on the contour of the hill.

We start once againby defining a class for adaline objects. The majority of the function is pretty much the same as the perceptron function. We just now have a cost matrix and are minimizing it, and we have an activation function that we use to call net_input.

import numpy as np 
import pandas as pd
import matplotlib.pyplot as plt 
df = pd.read_csv('https://archive.ics.uci.edu/ml/machine-learning-databases/iris/iris.data')

class AdalineGD(object):
    """
    Adaptive Linear Neuron classifier. 
    
    Parameters
    ------------
    
    eta : float 
        Learn rate (between 0.0 and 1.0)
    n_iter : int 
        Passes over the training dataset 
    
    Attributes 
    ------------
    
    w_ : 1d-array 
        Weights after fitting 
    errors_ : list 
        Number of misclassifications in every epoch 
    """
    
    def __init__(self, eta=0.01, n_iter = 50):
        self.eta = eta
        self.n_iter = n_iter 
        
    def fit(self, X, y): #loops over individual samples in training set and updates weight matrix. 
        """ fit training data 
        
        Parameters 
        X: {array-like}, shape = [n_samples, n_features]
        Training vectors, where n_samples is the number of samples and n_features is number of features 
        y: array-like, shape = [n_samples]
        Target values 
        
        Returns 
        -------
        self : object 
        """
        
        self.w_ = np.zeros(1 + X.shape[1]) #initialize weights matrix with zeros + the bias term 
        self.cost_ = [] #cost matrix. 
        
        for i in range(self.n_iter): #loop through all the epochs. At once, not individual by sample. 
            output = self.net_input(X)
            errors = (y - output)
            self.w_[1:] += self.eta * X.T.dot(errors)
            self.w_[0] += self.eta * errors.sum()
            cost = (errors**2).sum() / 2.0 #Residual errors squared. Term to minimize. 
            self.cost_.append(cost)
        
        return self
    
    def net_input(self,X):
        """ Calculate net input """
        
        return np.dot(X, self.w_[1:]) + self.w_[0] #activation function dot product. 
    
    def activation(self,X):
        "computer linear activation"
        return self.net_input(X)
    
    def predict(self,X):
        """Return class label after unit step """
        return np.where(self.net_input(X) >= 0.0, 1, -1)

Here we call a adaline object, and run it on the same iris dataset that we ran in the perceptron part. We try two different learning rates (Step sizes) and then plot the squared error against the number of iterations passthroughs. Usually picking the right learning rate is about seeing this type of plot converge.

Going back to Gradient Descent¶

The two graphs above display some of the issues with implementing gradient descent and the learning rate. The graph on the left side illustrates the problem with too large of a learning rate, as you can overshoot the global minimum.Variations re too large, the global minimum requires small iterations.Right side graph illustrates the problem with too small of a learning rate, takes a large amount of epochs to actually reach the minium.Illustrates need to feature scale for optimal performance. We're going to fix this below by using a feature scaling method called standardization, in order to give our data the property of a normal distribution i.e. mean centered at zero, with a stdev of 1.

from matplotlib.colors import ListedColormap
def plot_decision_regions(X,y,classifier, resolution=0.02):
    #setup marker generator and color map 
    
    markers = ('s', 'x', 'o', '^', 'v')
    colors = ('red', 'blue', 'lightgreen', 'gray', 'cyan')
    cmap = ListedColormap(colors[:len(np.unique(y))])

    #plot the decision surface 
    x1_min, x1_max = X[:, 0].min() - 1, X[:,0].max() + 1
    x2_min, x2_max = X[:, 1].min() - 1, X[:,1].max() + 1
    
    xx1, xx2 = np.meshgrid(np.arange(x1_min, x1_max, resolution), np.arange(x2_min, x2_max, resolution))
    
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1,xx2,Z,alpha=0.4, cmap=cmap)
    plt.xlim(xx1.min(), xx1.max())
    plt.ylim(xx2.min(), xx2.max())
    
    #plot class samples 
    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(x=X[y == cl, 0], y=X[y == cl, 1], alpha = 0.8, c=cmap(idx), marker = markers[idx], label=cl)
